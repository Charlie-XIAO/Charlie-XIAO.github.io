Problem 1.1 - Floating Point

Julia supports $16$, $32$, and $64$ bit floating point numbers (i.e., half, single, and double precision). See here for details. Write a code to calculate machine precision for each of these types.
Solution. The machine precision is reached when the machine can no longer tell $1.0$ from its next floating point number. Therefore, we write the following code for printing out the machine precision of specific data types. 
function machep(dtype)
    """Print out the machine precision of the specified data type."""
    for i = 1 : 100
        macheps = 2.0 ^ (-i)
        if (dtype(1.0) == dtype(1.0 + macheps))
            @printf("Machine precision for %s is %e\n", string(dtype), 2.0 ^ (-i + 1))
            break
        end
    end
end
We run this code with Float16, Float32, and Float64, respectively. 
machep(Float16)
machep(Float32)
machep(Float64)
This would produce the following output.

Machine precision for Float32 is 1.192093e-07
Machine precision for Float64 is 2.220446e-16

Problem 1.2 - IEEE Rounding

In the following questions, assume single precision format. See the text by Overton for reference, namely Chapter 5.
(a) What are the IEEE single format binary representations for the rounded value of $1/10$ using each of the four rounding modes (up, down, toward zero, and nearest)?
Solution. In the IEEE format, $1/10$ can be written in the normalized form as $$ \begin{align*} 0.1=(1.1001100110011001100110\underline{0}1100\cdots)_2\times2^{-4}, \end{align*} $$ where the underlined bit is the machine precision for single precision format. Therefore, under the four rounding modes, $1/10$ is respectively rounded to $$ \begin{align*} \texttt{round}(1/10)= \begin{cases} (1.10011001100110011001101)_2\times2^{-4}, & \text{round up}, \\ (1.10011001100110011001100)_2\times2^{-4}, & \text{round down}, \\ (1.10011001100110011001100)_2\times2^{-4}, & \text{round towards zero}, \\ (1.10011001100110011001101)_2\times2^{-4}, & \text{round to nearest}. \end{cases} \end{align*} $$
(b) What is $|\texttt{round}(1/10)-1/10|$, i.e., the absolute error, for each of the four rounding modes?
Solution. For the round up and the round to nearest modes, we can compute that $$ \begin{align*} \text{Absolute Error}={} & |\texttt{round}(1/10)-1/10|=(0.0000000000000000000000\underline{0}\overline{0011})_2\times2^{-4} \nonumber\\ ={} & (0.\overline{0011})_2\times2^{-27}=(0.\overline{1100})_2\times2^{-29}=(0.1)_{10}\times2^{-26}. \end{align*} $$ For the round down and the round towards zero modes, we can compute that $$ \begin{align*} \text{Absolute Error}={} & |\texttt{round}(1/10)-1/10|=(0.0000000000000000000000\underline{0}\overline{1100})_2\times2^{-4} \nonumber\\ ={} & (0.\overline{1100})_2\times2^{-27}=(0.1)_{10}\times2^{-24}. \end{align*} $$

Problem 1.3 - Multiple Root Finding

In this problem, you will write a code that will find all of the roots of the function $f(x)=x^5-3x^2+1$ inside the interval $[-2,2]$. For each part, please submit a printout of your code along with the values you obtain for the roots.
(a) Using the fact that the roots of $f$ on this interval are separated by at least $0.25$, write a program that implements the bisection algorithm to obtain estimates of the roots that are accurate to within $0.1$.
Solution. We implement the bisection method as follows. 
function bisection(func, left, right, tolerance)
    """Bisection method.

    Parameters
    ----------
    func : function
        The function to find a root for.
    left : Float64, etc.
        The left boundary of the interval to find a root.
    right : Float64, etc.
        The right boundary of the interval to find a root.
    tolerance : Float64, etc.
        The tolerance for the value of the root.

    Returns
    -------
    root : Float64 or boolean
        If a root is successfully found within tolerance, then it will be returned.
        Otherwise, the function returns false.
    """
    mid = (left + right) / 2
    mid_val = func(mid)
    if (abs(mid_val) < tolerance)
        return mid
    elseif (right - left < 1e-20)
        return false
    elseif (mid_val * func(left) < 0)
        return bisection(func, left, mid, tolerance)
    else
        return bisection(func, mid, right, tolerance)
    end
end

Solution. We implement the Newton's method as follows. 
function newton(f, f_, x0, tolerance, maxiter)
    """Newton's method.

    Parameters
    ----------
    f : function
        The function to find a root for.
    f_ : function
        The derivative of the function to find a root for.
    x0 : Float64, etc.
        The initial guess of the root, better near the real root.
    tolerance : Float64, etc.
        The tolerance for the value of the root.
    maxiter : Int64, etc.
        The maximum number of iterations to run Newton's method.

    Returns
    -------
    root : Float64
        The root found by Newton's method.
    """
    cur_x, cur_f = x0, f(x0)
    for _ = 1 : maxiter
        next_x = cur_x - cur_f / f_(cur_x)
        if (abs(next_x - cur_x) < tolerance)
            break
        end
        cur_x, cur_f = next_x, f(next_x)
    end
    return cur_x
end

Problem 1.4 - Newton Failures

Let the function $f$ be twice continuously differentiable in a neighborhood of $\xi$ (meaning that $f$, $f'$, and $f''$ are all continuous in a neighborhood of $\xi$). Furthermore, let $f(\xi)=f'(\xi)=0$ and $f''(\xi)>0$. The function $f$ is said to have a double root.
(a) At what rate, or in what range of rates, do you expect Newton's method to converge?
Solution. Quadratic convergence.
(b) Write a program to verify your estimated convergence behavior for the function $f(x)=(x-0.5)^2$, starting with the initial $x_0=0.25$. Turn in your code, as well as a table of values output by Newton's method.
Solution. In order to visualize the convergence behavior, we need to modify the Newton's method we implemented above to make it memorize the history. We implement the modified version as follows. 
function newton_hist(f, f_, x0, tolerance, maxiter)
    """Newton's method with iteration history.

    Parameters
    ----------
    f : function
        The function to find a root for.
    f_ : function
        The derivative of the function to find a root for.
    x0 : Float64, etc.
        The initial guess of the root, better near the real root.
    tolerance : Float64, etc.
        The tolerance for the value of the root.
    maxiter : Int64, etc.
        The maximum number of iterations to run Newton's method.

    Returns
    -------
    x_hist : 1D array
        The history of the x values during the Newton's method.
    f_list : 1D array
        The history of the f(x) values during the Newton's method.
    """
    x_hist, f_hist = [], []
    cur_x, cur_f = x0, f(x0)
    for _ = 1 : maxiter
        next_x = cur_x - cur_f / f_(cur_x)
        if (abs(next_x - cur_x) < tolerance)
            break
        end
        push!(x_hist, cur_x)
        push!(f_hist, cur_f)
        cur_x, cur_f = next_x, f(next_x)
    end
    return x_hist, f_hist
end;
