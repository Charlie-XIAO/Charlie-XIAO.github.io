sklearn.decomposition

FIX KernelPCA inverse transform when gamma is not given #26337
Originally, KernelPCA may produce incorrect results through inverse_transform if gamma=None. This was because gamma would be automatically set to 1 / n_features when the kernel is called. However, when using inverse_transform, n_features is not that of the data that the model is fitted on. I modified the logic so that now gamma will be chosen correctly in the very beginning of the fitting process. I also provided a new attribute gamma_ is provided for revealing the actual value of gamma used each time the kernel is called.

sklearn.ensemble

FIX Remove spurious feature names warning in IsolationForest #25931
When fitting an IsolationForest on a pandas dataframe with contamination != auto, the feature names were removed and a spurious warning was raised saying that 
X does not have valid feature names, but IsolationForest was fitted with feature names
This is because input data was validated when scoring the samples. To fix this, I created a private version of scoring function without input validation, and make the original public version validate the input and call this private one. Then when fitting, I used the private version so that feature names will not be removed and no warnings will be raised.

sklearn.feature_selection

FIX SequentialFeatureSelector throws IndexError when cv is a generator #25973
When fitting a SequentialFeatureSelector, the optional parameter cv would accept any iterable according to the documentation. However, a strange error will occur when passing cv as a generator: 
IndexError: list index out of range
This is because cv need to be reused during the fitting process, so I called `check_cv` to make sure that we have an iterator. Yet a conversion from a generator to an iterator is inefficient, so personally I still do not recommend using cv as generators.

sklearn.metrics

ENH RocCurveDisplay add option to plot chance level #25987
RocCurveDisplay plots the Receiver Operating Characteristic (ROC) curve. The ROC curve is constructed by plotting the true positive rate against the false positive rate, which measures the diagnostic ability of binary classifiers. The baseline is called chance level, which is the diagonal, and the farther the ROC curve is from the baseline, the better performance the classifier has. Therefore, I added a keyword plot_chance_level for plotting the chance level. I also provided a keyword chance_level_kw that supports a dictionary of matplotlib keywords for customizing the rendering of the chance level line.

sklearn.tree

FIX export_text and export_graphviz accepts feature and class names as array-like #26289
tree.export_text and tree.export_graphviz previously only accepts feature_names class_names as lists of strings. However, under many circumstances users get feature names and class names from numpy arrays or pandas dataframes, etc. I made these methods to support all array-like inputs for feature names and class names, so that users do not need to convert in advance.

Documentation Contributions

Items are sorted in reverse chronological order by the time of merge.

DOC fix SplineTransformer include_bias docstring #26018
The documentation of SplineTransformer wrote that if include_bias = True, then the last spline element inside the data range of a feature is dropped. However, the case is exactly the opposite, so I fixed the docstring.
