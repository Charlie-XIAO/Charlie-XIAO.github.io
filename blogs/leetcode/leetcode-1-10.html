Problem #1: Two Sum

Problem Statement: Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target. You may assume that each input would have exactly one solution, and you may not use the same element twice. You can return the answer in any order.

Example: 

Input: nums = [2, 7, 11, 15], target = 9
Output: [0, 1]

The intuitive solution of this problem is to traverse all possible pairs of numbers, and return their indices on the first match. Suppose there are $n$ numbers in total, then the total number of iterations would be $O(n^2)$, which leads to a quadratic runtime algorithm. The implementation can be as follows: 

def twoSum(nums: List[int], target: int) -> List[int]:
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return i, j

However, we can further solve this problem in linear runtime. To do this, we need to make use of the Python dictionaries, which support linear runtime operations. We will keep a dictionary with each key as the desired match of the integer at the index specified by the corresponding value. In other words, key = target - nums[value]. We traverse the given list only once. For each number, if it is not in the dictionary, we create a new key-value pair with value as its index and key as target minus itself. If it is in the dictionary, this means that it is the desired match of some number that we met earlier, and we are done. The implementation can be as follows: 

def twoSum(nums: List[int], target: int) -> List[int]:
    reqs = dict()
    for i in range(len(nums)):
        if nums[i] in reqs:
            return i, reqs[nums[i]]
        else:
            reqs[target - nums[i]] = i

Since Python dictionaries are built-in as hashmaps, k in d and getting items both take constant time, so this is a linear runtime algorithm, as desired. This simple example introduces the technique of hashmaps, which is widely used in many more scenarios to effectively reduce the runtime.



Problem #2: Add Two Numbers

Problem Statement: You are given two non-empty linked lists representing two non-negative integers. The digits are stored in reverse order, and each of their nodes contains a single digit. Add the two numbers and return the sum as a linked list. You may assume the two numbers do not contain any leading zero, except the number 0 itself.

Example: 

Input: l1 = 9 -> 9 -> 9 -> 9 -> 9 -> 9 -> 9, l2 = 9 -> 9 -> 9 -> 9
Output: 8 -> 9 -> 9 -> 9 -> 0 -> 0 -> 0 -> 1

This is a simple problem of linked lists. Literally how we do addition in reality, how we do for this problem. We align the digits (if one is shorter, we fill it to the same length as the other with zeros), and then add corresponding digits up. If the sum is greater than or equal to 10, we then keep its remainder and add an additional 1 to the next digit. The implementation can be as follows: 

# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

def addTwoNumbers(l1: Optional[ListNode], l2: Optional[ListNode]) -> Optional[ListNode]:
    head = ListNode()
    cur1, cur2, temp, prev = l1, l2, 0, head
    while cur1 is not None or cur2 is not None:
        if cur1 is None:
            val1 = 0
        else:
            val1 = cur1.val
            cur1 = cur1.next
        if cur2 is None:
            val2 = 0
        else:
            val2 = cur2.val
            cur2 = cur2.next
        new = ListNode(val=(val1 + val2 + temp) % 10)
        prev.next = new
        prev, temp = new, (val1 + val2 + temp) // 10
    if temp != 0:
        prev.next = ListNode(val=temp)
    return head.next



Problem #3: Longest Substring Without Repeating Characters

Problem Statement: Given a string s, find the length of the longest substring without repeating characters. Note that a substring must be contiguous, in contrast to subsequences which can be non-contiguous.

Example: 

Input: s = "pwwkew"
Output: 3

Intuitively, we can solve this problem by iterating through all possible starting and ending indices in the worst case. However, given the experience of solving Problem #1, we can again make use of Python dictionaries. Imagine how we tell the longest substring without repeating characters ourselves. For instance, if we have the string "helloworld", we will look through "hel", and when we see the next "l", we know that the longest substring without repeated characters before that is of length 3. Then we will look through "low", and when we see the next "o", we know that this substring cannot be any longer and thus the maximum length is still 3. However, this time we will not start from the second "o", but instead from the next character of the first "o", which is "w". As a simpler illustration for this, for the string "oabocde", there is no reason to ignore "ab" when we meet the second "o" since it is the first "o" that causes the repetition. Back to the example of "helloworld", by repeating the process described above we can obtain the longest substring. The only problem left is: how to we know if a character exists previously or not? To do this, we again use a dictionary to keep track of those characters and their last occurrence. In this way, we traverse the string only once, and the k in d operation for Python dictionaries runs constant time. This gives us a linear runtime algorithm, which can be implemented as follows: 

def lengthOfLongestSubstring(s: str) -> int:
    i, j, storage, maxlen = 0, 0, dict(), 0
    while j < len(s):
        # If the character exists previously in the part we are currently considering
        if s[j] in storage and storage[s[j]] >= i:
            maxlen = max(maxlen, j - i)
            i = storage[s[j]] + 1
        storage[s[j]] = j
        j += 1
    return max(maxlen, j - i)



Problem #4: Median of Two Sorted Arrays

Problem Statement: Given two sorted arrays nums1 and nums2 of size m and n respectively, return the median of the two sorted arrays. The overall run time complexity should be $O(\log(m+n))$.

Example: 

Input: nums1 = [1, 2], nums2 = [3, 4]
Output: 2.50000

This is without doubt a hard and interesting problem that I got stuck over a day. My first intuition was to merge the two sorted arrays in $O(m+n)$ time and then directly index the median. However, this is linear runtime while the problem requires a logarithm runtime solution. This inspires us that we may need an algorithm similar to binary search to solve this issue. But how exactly are we going to do this? We first generalize the statement to finding the $k$th smallest (assuming the arrays are sorted in ascending order) integer among the two arrays. To this end, we start by making some observations using the following example: 

nums1 = [1, 3, 5, 7, 9]
nums2 = [0, 2, 4, 6, 8]

Assume that we want to find the 7th smallest integer, which would be 6 in this case. We can consider nums1[3] = 7 and nums2[3] = 6. Since nums2[3] is smaller than nums1[3], it can be greater than at most 6 integers. This is because there are three integers smaller than it in nums2 and at most three integers smaller than it in nums1. As a result, if we want to find the 7th smallest integer in the two arrays, nums2[3] may be a candidate, but we do not need to consider nums2[0] to nums2[2] any more.

Now assume that we want to find the 6th smallest integer, which would be 5 in this case. We can consider nums1[2] = 5 and nums2[2] = 4. Since nums2[2] is smaller than nums1[2], it can be greater than at most 4 integers. This is because there are twp integers smaller than it in nums2 and at most two integers smaller than it in nums1. As a result, if we want to find the 6th smallest integer in the two arrays, all integers before nums2[2] in nums2, including itself, cannot be candidates.

To conclude the previous observations, if $k$ is odd, we look at index $\frac{k-1}{2}$ of both arrays. The first $\frac{k-1}{2}$ integers in the array with a smaller integer at this index can then be ignored. If $k$ is even, we look at index $\frac{k-2}{2}$ of both arrays. The first $\frac{k}{2}$ integers in the array with a smaller integer at this index can then be ignored. Putting these two cases together, we will look at index (k - 1) // 2 and ignore k // 2 integers in each step.

The algorithm can then be implemented recursively. There are several base cases. If the index we are looking for is already out of bound of one of the arrays, then we just directly take the $k$th integer in the other array. Moreover, if $k=1$, we will be ignoring nothing since 1 // 2 = 0, but we can directly return the smaller one between the first integers of the two arrays. As for the recursive calls, we keep two indices i and j to mark the first elements we are going to consider the arrays. That is, we consider only nums1[i:] and nums2[j:]. We also keep that k, which will decrement by k // 2 in each recursive call, until it becomes 1 in the worst case. This would give us an $O(\log k)$ algorithm, and since $k\leq m+n$, this would be an $O(\log(m+n))$ algorithm as desired. The implementation of this helper function can be as follows: 

def helper_kth(nums1, nums2, i, j, k):
    """The helper function for finding the kth smallest number in two lists.

    Parameters
    ----------
    nums1 : list[int]
        The first list.
    nums2 : list[int]
        The second list.
    i : int
        The starting index in the first list, i.e., we consider only nums1[i:].
    j : int
        The starting index in the second list, i.e., we consider only nums2[j:].
    k : int
        The kth smallest number to look for in nums1[i:] and nums2[j:].

    Returns
    -------
    res : int
        The kth smallest number in nums1[i:] and nums2[j:].
    """
    # Base case: if only one array is left for consideration
    len1, len2 = len(nums1), len(nums2)
    if i >= len1: return nums2[j + k - 1]
    if j >= len2: return nums1[i + k - 1]
    # Base case: if we are looking for the 1st
    if k == 1: return min(nums1[i], nums2[j])
    # Determine which array to cut
    target, cutsize = (k - 1) // 2, k // 2
    if i + target >= len1: cutFirst = False
    elif j + target >= len2 : cutFirst = True
    else: cutFirst = nums1[i + target] <= nums2[j + target]
    # Recursive step
    if cutFirst:
        return helper_kth(nums1, nums2, i + cutsize, j, k - cutsize)
    else:
        return helper_kth(nums1, nums2, i, j + cutsize, k - cutsize)

To find the median among a total of $m+n$ integers, we look for the $\frac{m+n+1}{2}$ th integer of them if $m+n$ is odd. Otherwise, we look for the average of the $\frac{m+n}{2}$ th and its next integer. This would lead to doing the following initial calls: 

def findMedianSortedArrays(self, nums1: List[int], nums2: List[int]) -> float:
    totlen = len(nums1) + len(nums2)
    if totlen % 2:
        return helper_kth(nums1, nums2, 0, 0, totlen // 2 + 1)
    else:
        return (helper_kth(nums1, nums2, 0, 0, totlen // 2) + helper_kth(nums1, nums2, 0, 0, totlen // 2 + 1)) / 2

However, note that we are performing two recursions if $m+n$ is even. Though this does not affect the asymptotic runtime, it does double the constant. To solve this problem, we can easily modify the helper so that it finds the $k$th and the $k+1$th integers at the meantime. The modification can be as follows: 

def helper_kth_and_next(nums1, nums2, i, j, k):
    # Base case: if only one array is left for consideration
    len1, len2 = len(nums1), len(nums2)
    if i >= len1: return nums2[j + k - 1], nums2[j + k]
    if j >= len2: return nums1[i + k - 1], nums1[i + k]
    # Base case: if we are looking for the 1st and 2nd
    if k == 1:
        if nums1[i] < nums2[j]:
            if i + 1 < len1: return nums1[i], min(nums1[i + 1], nums2[j])
            else: return nums1[i], nums2[j]
        else:
            if j + 1 < len2: return nums2[j], min(nums2[j + 1], nums1[i])
            else: return nums2[j], nums1[i]
    # Determine which array to cut
    target, cutsize = (k - 1) // 2, k // 2
    if i + target >= len1: cutFirst = False
    elif j + target >= len2 : cutFirst = True
    else: cutFirst = nums1[i + target] <= nums2[j + target]
    # Recursive step
    if cutFirst:
        return helper_kth_and_next(nums1, nums2, i + cutsize, j, k - cutsize)
    else:
        return helper_kth_and_next(nums1, nums2, i, j + cutsize, k - cutsize)

Intuitively this should be decreasing the runtime for $m+n$ odd by almost one half, but the final performance tested on LeetCode didn't change much. Regardless, the initial call for the even case would then be as follows: 

return sum(helper_kth_and_next(nums1, nums2, 0, 0, totlen // 2)) / 2

Problem #5: Longest Palindromic Substring

Problem Statement: Given a string s, return the longest palindromic substring in s. Note that a string is palindromic if it reads the same forward and backward.

Example: 

Input: s = "babad"
Output: "bab"
Explanation: "aba" is also a valid answer.

This is a very classical dynamic programming problem. Note that a substring is palindromic if and only if its first and last characters are the same, and the rest part is already palindromic. Therefore, we can start with substrings of length 1 or 2, and then to substrings of longer lengths. This dynamic programming problem runs quadratic time, because we will keep a matrix of size $n\times n$, with the $(i,j)$th entry representing whether the substring s[i:j] is palindromic. The naive implementation can be as follows: 

def longestPalindrome(self, s: str) -> str:
    n, l, r = len(s), 0, 1
    DP = [[0] * n for _ in range(n)]
    for i in range(n - 1):
        if s[i] == s[i + 1]:
            DP[i][i + 1], l, r = 1, i, i + 2
    for i in range(n - 2):
        if s[i] == s[i + 2]:
            DP[i][i + 2], l, r = 1, i, i + 3
    for k in range(3, n):
        for i in range(n - k):
            if s[i] == s[i + k] and DP[i + 1][i + k - 1]:
                DP[i][i + k], l, r = 1, i, i + k + 1
    return s[l:r]

However, this implementation is quite slow and memory consuming. It runs 3 seconds for the largest test case on LeetCode, and uses more than 20000 KB of memory space. In order to improve its performance, we make the following modifications.

  • Since substrings of length $k$ only depend on substrings of length $k-2$, we can keep just two lists instead of an array. This reduces the memory space from $O(n^2)$ to $O(2n)$.
  • Whenever the substrings of some odd length $k$ is all non-palindromic, the substrings of odd lengths greater than $k$ must all be non-palindromic. This also applies to substrings of even lengths. Therefore, we may cut down a large number of iterations for certain input strings.

The implementation can be as follows: 

def longestPalindrome(self, s: str) -> str:
    n, l, r = len(s), 0, 1
    oddbrk, evenbrk = True, True
    oddlst, evenlst = [0] * n, [0] * n
    # Palindromes of length 2
    for i in range(n - 1):
        if s[i] == s[i + 1]:
            evenlst[i], l, r, evenbrk = 1, i, i + 2, False
    # Palindromes of length 3
    for i in range(n - 2):
        if s[i] == s[i + 2]:
            oddlst[i], l, r, oddbrk = 1, i, i + 3, False
    # Palindromes of length >= 4
    for k in range(3, n):
        if k % 2:
            if evenbrk: continue
            else:
                evenbrk = True
                for i in range(n - k):
                    if s[i] == s[i + k] and evenlst[i + 1]:
                        evenlst[i], l, r, evenbrk = 1, i, i + k + 1, False
                    else:
                        evenlst[i] = 0
        else:
            if oddbrk: continue
            else:
                oddbrk = True
                for i in range(n - k):
                    if s[i] == s[i + k] and oddlst[i + 1]:
                        oddlst[i], l, r, oddbrk = 1, i, i + k + 1, False
                    else:
                        oddlst[i] = 0
    return s[l:r]

This implementation reduces the runtime to approximately 1 second and memory consumption to approximately 14000 KB. However, it is quite tedious, and though I made the previous optimizations, it still beats only 40% of Python submissions. I'm not sure what I did wrong, but the implementation above is the best I can do at this moment.



Problem #6: Zigzag Conversion

Problem Statement: The string "PAYPALISHIRING" is written in a zigzag pattern on a given number of rows like this: (you may want to display this pattern in a fixed font for better legibility) 

P   A   H   N
A P L S I I G
Y   I   R

And then read line by line: "PAHNAPLSIIGYIR". Write the code that will take a string and make this conversion given a number of rows.

Example: 

Input: s = "PAYPALISHIRING", numRows = 3
Output: "PAHNAPLSIIGYIR"

This is in fact a math problem. Let us consider the same example as above, but with indices: 

 0     4     8    12
 1  3  5  7  9 11 13
 2     6    10

In the first line, we have indices 4 * i. In the last line, we have indices 4 * i + 2. In the lines in the middle, for instance the $k$th line (counting from 0), we have indices 4 * i + k. Moreover, in between each two of those indices, we furthermore have indices 4 * (i + 1) - k. Note that corner case that the number of rows is 1. In this case, we just directly return the original string. These observations lead to the implementation as follows: 

def convert(s: str, numRows: int) -> str:
    if numRows == 1:
        return s
    storage, subsize = [], 2 * (numRows - 1)
    # The first row
    storage.extend([s[subsize * i] for i in range((len(s) - 1) // subsize + 1)])
    # The rows in the middle
    for k in range(1, numRows - 1):
        j = k
        while j < len(s):
            storage.append(s[j])
            intval = j + subsize - 2 * k
            if intval < len(s):
                storage.append(s[intval])
            else: break
            j += subsize
    # The last row
    storage.extend([s[subsize * i + numRows - 1] for i in range((len(s) - numRows) // subsize + 1)])
    return "".join(storage)



Problem #7: Reverse Integer

Problem Statement: Given a signed 32-bit integer x, return x with its digits reversed. If reversing x causes the value to go outside the signed 32-bit integer range $[-2^{31},2^{31}-1]$, then return 0. Note that one would also need to handle negative integers.

Example: 

Input: x = 123
Output: 321

This is again a math problem. Let us consider the same example as above, then by x // 10 we can obtain the digit 3 from the remainder. Now x would become 12, and we repeat the previous process to obtain the digit 2, and then the digit 1. Note that the 3 needs to be multiplied by 100 in the result, 2 by 10, and 1 by 1. This means that if in each iteration, we multiply the previous result by 10 and add the new remainder, we can directly get the final reversed integer. To be more specific, in the first iteration we get 3, then in the next iteration we get 3*10+2, and then in the final iteration we get (3*10+2)*10+1. However, there are still two corner cases that we haven't considered. If the given number is negative, we deal with its absolute value as above and finally reverse the sign. Moreover, if the reversed number exceeds the signed 32-bit integer range, we return 0. The implementation can be as follows: 

def reverse(x: int) -> int:
    res, target = 0, abs(x)
    while target > 0:
        res = res * 10 + target % 10
        target //= 10
    res = res * (1 if x > 0 else -1)
    return res if res <= 2 ** 31 - 1 and res >= -2 ** 31 else 0



Problem #8: String to Integer (atoi)

This is a really tedious problem and I'm not going to specify on it. Here is the link to the description of this problem for your reference. A possible implementation can be as follows: 

def myAtoi(s: str) -> int:
    indenting, start, end = True, -1, len(s)
    for i in range(len(s)):
        ch = s[i]
        if indenting and ch != " ":
            indenting = False
        if not indenting:
            # If already reading but meet a non-digit
            if start != -1 and not ch.isdigit():
                end = i
                break
            # If not reading and meet and starting character
            elif start == -1 and (ch.isdigit() or ch in "+-"):
                start = i
            # If not reading but character is invalid 
            elif start == -1:
                end = i
                break
    # If never started reading
    if start == -1: res = 0
    # Corner cases such as +-123
    elif end - start == 1 and s[start] in "+-": res = 0
    else: res = int(s[start:end])
    # Clamp the result
    if res > 2 ** 31 - 1: return 2 ** 31 - 1
    elif res < -2 ** 31: return -2 ** 31
    else: return res



Problem #9: Palindrome Integer

Problem Statement: Given an integer x, return true if x is a palindrome, and false otherwise. Note that an integer is a palindrome when it reads the same forward and backward.

Example: 

Input: x = 121
Output: true
Input: x = -121
Output: false

This is another dull problem. Python and most other programming languages provide functions or methods for converting integers into strings. Then simply by comparing the string and its reversed version we can obtain the result. Indeed the problem asked for possible solutions without converting the integers to strings. To do this, we can just reverse the integer mathematically and compare with the original number. That being said, this method would be much slower. The implementation of the former method is as follows: 

def isPalindrome(x: int) -> bool:
    if x < 0:
        return False
    s = str(x)
    return s == s[::-1]



Problem #10: Regular Expression Matching

Problem Statement: Given an input string s and a pattern p, implement regular expression (regex) matching with support for "." and "*", where "." matches any single character, and "*" matches zero or more of the preceding element. The matching should cover the entire input string.

Example: 

Input: s = "aaabcaab", p = "a*b.a*b"
Output: True

This is a relatively hard dynamic programming problem. Its difficulty lies in the complexity of the mechanism of regex (even it is simplified in this problem), making it hard to determine subproblems and recursive relations. Consider the dynamic programming matrix DP of size $(n+1)\times(m+1)$, where $n$ is the length of the string and $m$ is the length of the regex. Moreover, let DP[i][j] represent if the first $i$ characters of the string and the first $j$ characters of the regex form a match. Using the same example as provided, this will give us the following matrix: 

   0  a  *  b  .  a  *  b
0  1  0  1  0  0  0  0  0
a  0  1  1  0  0  0  0  0
a  0  0  1  0  0  0  0  0
a  0  0  1  0  0  0  0  0
b  0  0  0  1  0  0  0  0
c  0  0  0  0  1  0  1  0
a  0  0  0  0  0  1  1  0
a  0  0  0  0  0  0  1  0
b  0  0  0  0  0  0  0  1

We need to first discuss the base cases. The first column of DP represents whether a substring matches an empty regex. Clearly, nothing except an empty string can match an empty regex, so only DP[0][0] = 1 in the first column. Next, let us look at the first row, which represents whether an empty string matches some prefix of the regex.

  • Clearly, an empty string does not match "a".
  • An empty string matches "a*" since it can stand for zero occurrences.
  • An empty string does not match "a*b" since it does not have the character "b".
  • etc.

In other words, if the last character in the current regex is "*", an empty string would match it if and only if the empty string matches it without the last "*" and its preceding character. If the last character in the current regex is not "*", then an empty string cannot match it since the empty string does not have any single character.

Next we iterate row by row to fill in the DP matrix. Denote the current regex we are looking at by cur_p, and the current string by cur_s, then there are mainly to cases.

Case 1: The last character of cur_s is "*". In this case, if cur_s matches cur_p[:-2], this would imply that cur_s matches cur_p as well since any character followed by an asterick can represent zero occurrence. For instance, let us look at DP[5][6] in our example. The string "aaabc" matches the regex "a*b.a*" simply because it matches the regex "a*b.", as is shown by DP[5][4]. However, this is not the only circumstance where cur_s can match cur_p. For instance, DP[6][6] shows that the string "aaabca" matches the string "a*b.a*", but not because of the previous reason. The reason for this match is because "aaabc" matches "a*b.a*", and further more the last character "a" in cur_s is the same as the one specified before the asterick in cur_p. Therefore, now we can conclude the following recurrence relation: $$ \texttt{DP}[i][j] = \begin{cases} 1, & \text{if $\texttt{DP}[i][j-2]=1$}, \\ 1, & \text{if $\texttt{DP}[i-1][j]=1$ and $\texttt{cur_s}[-1]=\texttt{cur_p}[-2]$,} \\ 0, & \text{otherwise}, \end{cases} \quad\text{if $\texttt{cur_p}[-1]=\texttt{"*"}$}. $$

Case 2: The last character of cur_s is not "*". In this case, take DP[5][4] as an example, which shows that the string "aaabc" and the "a*b." form a match. This is literally because "c" and "." are the same, and the rest part "aaab" matches "a*b", as is shown by DP[4][3]. In other words, cur_s matches cur_p if and only if cur_s[-1] and cur_p[-1] are the same, while cur_s[:-1] matches cur_p[:-1] in this case. This leads us to the following recurrence relation: $$ \texttt{DP}[i][j] = \begin{cases} 1, & \text{if $\texttt{DP}[i-1][j-1]=1$ and $\texttt{cur_s}[-1]=\texttt{cur_p}[-1]$,}, \\ 0, & \text{otherwise}, \end{cases} \quad\text{if $\texttt{cur_p}[-1]\neq\texttt{"*"}$}. $$

With the base cases and recurrence relations discussed as above, we can implement the simplified regex matching algorithm as follows: 

def isMatch(s: str, p: str) -> bool:
    n, m = len(s), len(p)
    DP = [[0] * (m + 1) for _ in range(n + 1)]
    # The empty regex only matches an empty string
    DP[0][0] = 1
    # The empty string matches a regex if all characters are followed by *
    for j in range(1, m + 1):
        if p[j - 1] == "*" and DP[0][j - 2]:
            DP[0][j] = 1
    # Fill in the DP matrix row by row
    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if p[j - 1] == "*":
                if DP[i][j - 2] or (DP[i - 1][j] and (p[j - 2] == "." or p[j - 2] == s[i - 1])):
                    DP[i][j] = 1
            else:
                if DP[i - 1][j - 1] and (p[j - 1] == "." or p[j - 1] == s[i - 1]):
                    DP[i][j] = 1
    return bool(DP[n][m])
