Problem #1: Two Sum

Problem Statement: Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target. You may assume that each input would have exactly one solution, and you may not use the same element twice. You can return the answer in any order.

Example: 

Input: nums = [2, 7, 11, 15], target = 9
Output: [0, 1]

The intuitive solution of this problem is to traverse all possible pairs of numbers, and return their indices on the first match. Suppose there are $n$ numbers in total, then the total number of iterations would be $O(n^2)$, which leads to a quadratic runtime algorithm. The implementation can be as follows: 

def twoSum(nums: List[int], target: int) -> List[int]:
    for i in range(len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[i] + nums[j] == target:
                return i, j

However, we can further solve this problem in linear runtime. To do this, we need to make use of the Python dictionaries, which support linear runtime operations. We will keep a dictionary with each key as the desired match of the integer at the index specified by the corresponding value. In other words, key = target - nums[value]. We traverse the given list only once. For each number, if it is not in the dictionary, we create a new key-value pair with value as its index and key as target minus itself. If it is in the dictionary, this means that it is the desired match of some number that we met earlier, and we are done. The implementation can be as follows: 

def twoSum(nums: List[int], target: int) -> List[int]:
    reqs = dict()
    for i in range(len(nums)):
        if nums[i] in reqs:
            return i, reqs[nums[i]]
        else:
            reqs[target - nums[i]] = i
Since Python dictionaries are built-in as hashmaps, k in d and getting items both take constant time, so this is a linear runtime algorithm, as desired. This simple example introduces the technique of hashmaps, which is widely used in many more scenarios to effectively reduce the runtime.

Problem #2: Add Two Numbers

Problem Statement: You are given two non-empty linked lists representing two non-negative integers. The digits are stored in reverse order, and each of their nodes contains a single digit. Add the two numbers and return the sum as a linked list. You may assume the two numbers do not contain any leading zero, except the number 0 itself.

Example: 

Input: l1 = 9 -> 9 -> 9 -> 9 -> 9 -> 9 -> 9, l2 = 9 -> 9 -> 9 -> 9
Output: 8 -> 9 -> 9 -> 9 -> 0 -> 0 -> 0 -> 1

This is a simple problem of linked lists. Literally how we do addition in reality, how we do for this problem. We align the digits (if one is shorter, we fill it to the same length as the other with zeros), and then add corresponding digits up. If the sum is greater than or equal to 10, we then keep its remainder and add an additional 1 to the next digit. The implementation can be as follows: 

# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

def addTwoNumbers(l1: Optional[ListNode], l2: Optional[ListNode]) -> Optional[ListNode]:
    head = ListNode()
    cur1, cur2, temp, prev = l1, l2, 0, head
    while cur1 is not None or cur2 is not None:
        if cur1 is None:
            val1 = 0
        else:
            val1 = cur1.val
            cur1 = cur1.next
        if cur2 is None:
            val2 = 0
        else:
            val2 = cur2.val
            cur2 = cur2.next
        new = ListNode(val=(val1 + val2 + temp) % 10)
        prev.next = new
        prev, temp = new, (val1 + val2 + temp) // 10
    if temp != 0:
        prev.next = ListNode(val=temp)
    return head.next

Problem #3: Longest Substring Without Repeating Characters

Problem Statement: Given a string s, find the length of the longest substring without repeating characters. Note that a substring must be contiguous, in contrast to subsequences which can be non-contiguous.

Example: 

Input: s = "pwwkew"
Output: 3

Intuitively, we can solve this problem by iterating through all possible starting and ending indices in the worst case. However, given the experience of solving Problem #1, we can again make use of Python dictionaries. Imagine how we tell the longest substring without repeating characters ourselves. For instance, if we have the string "helloworld", we will look through "hel", and when we see the next "l", we know that the longest substring without repeated characters before that is of length 3. Then we will look through "low", and when we see the next "o", we know that this substring cannot be any longer and thus the maximum length is still 3. However, this time we will not start from the second "o", but instead from the next character of the first "o", which is "w". As a simpler illustration for this, for the string "oabocde", there is no reason to ignore "ab" when we meet the second "o" since it is the first "o" that causes the repetition. Back to the example of "helloworld", by repeating the process described above we can obtain the longest substring. The only problem left is: how to we know if a character exists previously or not? To do this, we again use a dictionary to keep track of those characters and their last occurrence. In this way, we traverse the string only once, and the k in d operation for Python dictionaries runs constant time. This gives us a linear runtime algorithm, which can be implemented as follows: 

def lengthOfLongestSubstring(s: str) -> int:
    i, j, storage, maxlen = 0, 0, dict(), 0
    while j < len(s):
        # If the character exists previously in the part we are currently considering
        if s[j] in storage and storage[s[j]] >= i:
            maxlen = max(maxlen, j - i)
            i = storage[s[j]] + 1
        storage[s[j]] = j
        j += 1
    return max(maxlen, j - i)

Problem #5: Longest Palindromic Substring

Problem Statement: Given a string s, return the longest palindromic substring in s. Note that a string is palindromic if it reads the same forward and backward.

Example: 

Input: s = "babad"
Output: "bab"
Explanation: "aba" is also a valid answer.

This is a very classical dynamic programming problem. Note that a substring is palindromic if and only if its first and last characters are the same, and the rest part is already palindromic. Therefore, we can start with substrings of length 1 or 2, and then to substrings of longer lengths. This dynamic programming problem runs quadratic time, because we will keep a matrix of size $n\times n$, with the $(i,j)$th entry representing whether the substring s[i:j] is palindromic. The naive implementation can be as follows: 

def longestPalindrome(self, s: str) -> str:
    n, l, r = len(s), 0, 1
    DP = [[0] * n for _ in range(n)]
    for i in range(n - 1):
        if s[i] == s[i + 1]:
            DP[i][i + 1], l, r = 1, i, i + 2
    for i in range(n - 2):
        if s[i] == s[i + 2]:
            DP[i][i + 2], l, r = 1, i, i + 3
    for k in range(3, n):
        for i in range(n - k):
            if s[i] == s[i + k] and DP[i + 1][i + k - 1]:
                DP[i][i + k], l, r = 1, i, i + k + 1
    return s[l:r]
However, this implementation is quite slow and memory consuming. It runs 3 seconds for the largest test case on LeetCode, and uses more than 20000 KB of memory space. In order to improve its performance, we make the following modifications.

  • Since substrings of length $k$ only depend on substrings of length $k-2$, we can keep just two lists instead of an array. This reduces the memory space from $O(n^2)$ to $O(2n)$.
  • Whenever the substrings of some odd length $k$ is all non-palindromic, the substrings of odd lengths greater than $k$ must all be non-palindromic. This also applies to substrings of even lengths. Therefore, we may cut down a large number of iterations for certain input strings.
The implementation can be as follows: 
def longestPalindrome(self, s: str) -> str:
    n, l, r = len(s), 0, 1
    oddbrk, evenbrk = True, True
    oddlst, evenlst = [0] * n, [0] * n
    # Palindromes of length 2
    for i in range(n - 1):
        if s[i] == s[i + 1]:
            evenlst[i], l, r, evenbrk = 1, i, i + 2, False
    # Palindromes of length 3
    for i in range(n - 2):
        if s[i] == s[i + 2]:
            oddlst[i], l, r, oddbrk = 1, i, i + 3, False
    # Palindromes of length >= 4
    for k in range(3, n):
        if k % 2:
            if evenbrk: continue
            else:
                evenbrk = True
                for i in range(n - k):
                    if s[i] == s[i + k] and evenlst[i + 1]:
                        evenlst[i], l, r, evenbrk = 1, i, i + k + 1, False
                    else:
                        evenlst[i] = 0
        else:
            if oddbrk: continue
            else:
                oddbrk = True
                for i in range(n - k):
                    if s[i] == s[i + k] and oddlst[i + 1]:
                        oddlst[i], l, r, oddbrk = 1, i, i + k + 1, False
                    else:
                        oddlst[i] = 0
    return s[l:r]
This implementation reduces the runtime to approximately 1 second and memory consumption to approximately 14000 KB. However, it is quite tedious, and though I made the previous optimizations, it still beats only 40% of Python submissions.
